Basic Concepts

Git: is a Software for source code control (cloud).

Github: is an online service (website) which hosts Git. It is a popular open source social network for coding community.

Install Git

To have version control for the code using github, you need to download git, as github is a web interface built around a VCS (git).

It is open source, and FREE.

Version Control System (VCS)

VCS tracks and control change during development and maintainenace of code (electronic documents). It allows software developers to work together and sometimes simultaneouely without overwriting each other's changes while maintaining the history of every version.

Git Operations

Create an account on Github or sign in to Github

How to create (new) repositories/files

In github website

  • At your dashboard: Click New button. Name your repository (file/folder). It creates a new repository in github.

  • By forking: Click Fork button from existing repository. It creates your own copy of someone else's repository

  • By cloning: Copy the HTTPS web url and paste it into the command line Terminal. This will be your local repository.

  • By downloading: Click the Download button to Open in your computer or Download Zip. This will be the local repository to your computer.

  • On github repository:

    • Create New File above the file lists. Name it: fileName.extension
    • Add content on the Edit new file tab
    • Preview, Commit msg, Email address, Propose new file

Adding (new) file to a repositories

Using Command Line

  • By creating a new file, then upload and Commit 

      cd ~/Desktop/directoryName
        git init    
        git checkout -b develop 
        touch fileName.md  //make an empty file

This will create a new git repo to manage the current folder.

Edit this fileName by adding line(s) and save it.

But we want to save this remote. Copy its HTTPS / SSH clone url which looks like git@github.com:UserName/repoName.git or https://github.com/UserName/repoName.git

Then add the remote

    git remote add origin <url>

Finally, push the existing repo

    git push -u origin master

  • With an existing file in your computer desktop: Upload & Commit 

    echo "# repoName" >> fileName
        git init
        git add fileName //commits the file
        git commit -m "first commit" //adds the file
        git remote add origin <github repo url>
        git push -u origin master
        /*git commit -am <msg>*/is add & commit together

    Origin is remote-server name (remote repository), while master is just a branch-name. Master is the default branch name of the current working file (the repository).

  • Drag and Drop

Drag and drop folder or files on to the file tree in github.

  • Upload

At the main page of the repository, click Upload files under your repository name.

Make some changes (update files)

  • By cloning an existing repo on your desktop.

Clone mirrors whole repository. The remote repo will be downloaded and copied in your local repo. You can perform operations using this local repository. You do not need network to do this, until during synchronization.

    cd ~/Desktop
            git clone <git-repo-url> directoryName
            cd directoryName
            git checkout -b develop //to update the dev 
            git checkout -b myFeatureBranch //create branch

It will also add a remote named 'origin' back to source

This will create a folder named directoryName and clones the repo there and adds a remote named "origin" back to the source. 'remote' is automatically created when cloning.

If you leave the name part out, it will clone to the folder with the repo original name

Do NOT commit directly to master-branch NOR commit directly to the develop-branch.

Instead, CREATE feature-branches for unit issues

After making some changes 

git add .  #commit this revision changes .The add-dot stages the current directory and all the files and dir it contains except for empty directories.
        git commit -m <message explaining changes> #saving/adds

Merging the change.

git checkout develop
        git merge -- myFeatureBranch
        git push origin develop //stores changes to Git repo

This will add those features/edits to the develop branch.

Merge Conflicts 

 git pull

Pull copies changes from remote repo to local one.

Project Release 

git checkout master  //feature branch created 
        git merge -- develop  //update to develop branch
        git push origin master  
        git tag v1.0   //tag is name with a release version
        git push origin v1.0.0

This will add the develop branch total features to the master in the final release.

Push copies changes from local to remote repo to store permanently in Git repo. It is used for synchronization between two repos.

NOTE: After merging with master branch you can delete develop branch

Most of these operations (commit, branches creation, etc can be performed offline. Internet is required only when publishing changes (push) or taking latest changes (pull).

Deleting a Repo

In the repo dashboard, click setting, scroll to the bottom, and delete. Make sure of this before doing so, as you can't recover the deleted repo.
